# SAFETY DISCLAIMER — SmartSalai Edge-Sentinel

> **READ BEFORE USE — MANDATORY**

---

## ⚠️ NOT AN ADAS SYSTEM — NOT CERTIFIED FOR VEHICLE OPERATION

**SmartSalai Edge-Sentinel is a RESEARCH PROTOTYPE only.**

This software:

- **Has NOT been validated against any automotive safety standard** (ISO 26262, SOTIF / ISO 21448, AIS-145, UN R155 / R156, or any equivalent).
- **Has NOT undergone a Hazard Analysis and Risk Assessment (HARA)** or any formal safety engineering process.
- **Has NOT been tested on a certified hardware-in-the-loop (HiL) or vehicle-in-the-loop (ViL) platform**.
- **Must NOT be used to make driving decisions in any vehicle** — civilian car, two-wheeler, school bus, ambulance, fire-truck, or any other vehicle type.
- **Must NOT be trusted to detect, classify, or respond to road hazards in real time** in a way that influences the operation of a moving vehicle.
- **Does NOT guarantee correctness, accuracy, or reliability** of any alert, detection, or legal recommendation produced.
- **Is NOT compliant** with MoRTH AIS-140 VLTD requirements for production deployment.

---

## 🚫 Prohibited Uses

The following uses are **strictly prohibited** without additional formal safety engineering and regulatory approval:

| Prohibited Use | Reason |
|---|---|
| Controlling throttle, steering, or braking | Unvalidated actuator control — life-critical |
| Generating driver alerts during active driving without driver acknowledgement | Risk of distraction / alert fatigue |
| Claiming false-positive-free blind-spot detection | Model is unvalidated on production sensor data |
| Deploying in emergency vehicles (ambulance, fire-truck, police) | No certified functional safety evidence |
| Deploying in school transport or public transit | Passenger safety requires AIS-145 certification |
| Selling or distributing as a certified safety product | Illegal under Motor Vehicles Act 1988/2019 rules |

---

## ✅ Permitted (Research / Simulation) Uses

- **Offline simulation and replay analysis** of recorded dashcam footage.
- **Research evaluation** of detection algorithms against labelled datasets.
- **Academic hackathon demonstration** with explicit on-screen safety disclaimer.
- **Driver logging assistant** (non-real-time post-trip analysis).
- **Legal document generation** reviewed by a licensed advocate before submission.

---

## 🧑‍💻 Developer Responsibilities

All contributors and users of this codebase agree that:

1. **The driver is always responsible** for vehicle operation. Software cannot legally or ethically substitute for human judgment in a safety-critical context.
2. Any **alerts or recommendations** produced by this system are **informational only** and must be accompanied by a disclaimer in the user interface.
3. Any **legal document** produced by the Section 208 drafter must be reviewed by a qualified advocate before submission.
4. **Model outputs are probabilistic** — false positives and false negatives will occur.
5. **Performance on laptops (Acer Aspire 7 / RTX 3050)** does not guarantee equivalent performance on embedded/NPU hardware.

## 🛡️ Guardrails Built into This Codebase

| Guardrail | Location | Behaviour |
|---|---|---|
| Mock-mode fallback | `agents/sign_auditor.py` | If ONNX model is absent, returns advisory-only mock detections |
| Confidence threshold gating | `agents/sec208_drafter.py` | Challenges only issued at ≥0.85 confidence |
| Fail-closed on missing model | `agents/imu_near_miss_detector.py` | Deterministic fallback — no phantom alerts |
| Privacy: GPS ZKP envelope | `core/zkp_envelope.py` | All GPS emits are Pedersen-committed; no plaintext GPS in telemetry |
| No vehicle actuation code | Entire codebase | Zero steering/throttle/braking API calls |
| Advisory-only alert text | `agents/acoustic_ui.py` | All phrases include "possible" / "check" qualifiers |

## 📋 Regulatory Context (India)

- **Motor Vehicles Act 1988 as amended by MVA 2019**: governs vehicle equipment approvals.
- **AIS-140**: regulates VLTD (Vehicle Location & Tracking Devices) approved for public transport. This codebase does not claim AIS-140 approval.
- **MoRTH iRAD schema**: used for telemetry structure only. iRAD records generated by this system are simulation artefacts and must not be submitted as official crash reports.
- **Information Technology Act 2000 (Section 43A)**: personal data (GPS, video) processed by this system requires appropriate privacy safeguards before production deployment.
